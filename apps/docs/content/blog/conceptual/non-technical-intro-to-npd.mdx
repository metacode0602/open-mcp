## 这本书适合谁？

新产品开发（NPD）是将新产品、服务或技术推向市场的过程。无论是初次创业者还是大型企业，将新产品推向市场都是每个愿景的一部分。在本书中，我们特别关注“互联网产品”——进一步提炼，即网络和移动应用程序。

本书的主要目标是提供足够的介绍，让读者了解产品开发和推向市场所需的一切。它适用于参与此生命周期的每个人。

我们将把产品推向市场称为“进入市场”（GTM）。GTM 是一种策略，它利用你的优势来推出产品。许多人参与其中，例如：

- 领导层：他们希望抓住机会，根据团队的成熟度，讨论有时会涉及 TAM 等术语或仅仅是直觉。尽管如此，机会是存在的。
- 产品团队：产品团队负责理解客户需求和痛点，并制定解决方案来解决这些问题。所有这些都与商业计划相关联。这也包括负责 UX/UI 的设计团队。
- 工程团队：技术团队负责开发、测试和处理规模问题。他们还测试应用程序，将其部署到云端并运行。他们也负责维护。

我们可以进一步细分以上类别，但这不会为我们的讨论增加更多价值。本书首先面向领导层和产品团队，但工程团队也应该阅读，否则他们怎么会知道领导层和产品团队真正了解什么！

至少通过这本书，每个人都能以最基本的方式了解其他人知道什么。

## 我们是如何写这本书的？

我们从一开始的目标就是让这本书简短。我们在 Goodreads 上看到大量评论，说有些内容应该是一篇博客，或者至少比应有的篇幅小 1/3。我们的目标是让你一次读完这本书。

快速浏览概念的挑战在于学习曲线以及我们应该对读者做出的假设。我们的努力是保持学习曲线平缓，并且不会完全误解某些内容。有时我们使用手绘图片来唤醒你内心的童真，有时使用过度简化的比喻。

但我们确信，读了这本书总比不读好。如果你已经了解所有这些概念，你可以给我们反馈，并改善地球上其他人的生活。

我们尽量让每个章节最多 1-2 页，并且应该快速结束。所以，为了结束本章——这本书不是程序性的，也没有要遵循的步骤。它是对某些概念的介绍，有些信息是类比的——这意味着你将用在这里学到的知识扩展你的知识。

最后，你会在大脑中形成这些概念与你自身知识相连接的映射。

## 什么是应用程序？

以下是我们所说的应用程序：

- 移动应用：你从 Android 的 Play 商店或 iOS 的 App Store 安装的东西。你通常按名称搜索，然后点击“安装”。
- 网络应用：你使用 URL 在浏览器中打开的东西。浏览器可以在手机和笔记本电脑上运行。浏览器的例子包括 Chrome、Firefox、Edge 等。
- 桌面应用：你从 Macbook 或 Windows 的应用商店安装在电脑上的东西。

一个观点：如果我们尝试泛化，任何东西都可以是一个应用程序。但我们只将上述内容称为应用程序或 App。它是功能性的，是动态的，它响应用户输入。

那么网站是什么呢？

通常网站是静态的，也就是说它们不会改变，它们是信息性的，它们是内容。网站和网络应用可能看起来相似——但用途不同。你的用户可能首先登陆你的网站，然后注册/登录，然后进入他们应该探索和使用的网络应用。

我们可以称之为“Amazon.com”！

有些产品只有移动应用和网站（没有网络应用）。

## 前端和后端

应用可以是网络或移动应用，这些通常被称为“前端”。可以把它想象成“面向”用户的东西，所以你的客户主要与它们互动。洗衣机上的按钮就是它的前端。

为了使这些应用程序工作——它们要么运行在某人的手机上，要么运行在网络浏览器中——当它们运行时，有一些关键操作使这些应用程序具有功能性：

1. 他们正在查看、探索和过滤的数据
2. 他们能够进行的预订、订购或仅仅是保存笔记的操作

产品的功能是你提供给客户的，用户通过其前端使用该功能，而该功能由后端提供支持。

你只需要记住的一个主要概念是：后端不是运行在手机或网络浏览器中，它运行在某个服务器上。为了简单起见，我们将这个服务器称为“云”（你可能听说过 AWS、Azure、Google Cloud 等，但还有很多其他的）。

总结一下：前端与后端交互，后端运行在云端！

## 前端和后端如何交互？

把前端和后端想象成两栋房子，两栋房子都有电话，这些电话通过电线连接。它们就是这样交互的吗？电话不总是从你的房子直接连接到你朋友的房子，但你的声音通过各种路径到达你朋友那里。前端和后端也通过互联网交互。

## 我们做什么？

我们从 2011 年左右开始开发网络和移动应用程序，为我们的几个朋友和他们的联系人做过自由职业。在最初的 4 年里，我们开发了解决制造业、建筑业、体育、娱乐等领域问题的应用程序。那时我们只有 2 名开发者。很快我们意识到一个问题——非技术出身的“领域专家”需要一个能够给予关注、承诺和热情，与他们一起构建想法、发现新问题的团队。他们另一个选择是组建一个技术团队，这有一个很大的先决条件，即找到一个联合创始人级别的技术人员。

2016 年，我们成立了 Betalectic，以更大规模地解决这些问题。在解决了不同领域多种类型的问题后，我们可以说，我们做的是：

- 我们设计和构建全栈网络和移动应用程序。这包括前端、后端，并将它们部署到云端。
- 我们长期（想想几年）维护和支持这些应用程序。
- 我们构建仅 API 的解决方案。
- 我们参与端到端产品开发生命周期（PDLC），我们将在另一章中定义它。
- 我们帮助将想法转化为业务。
